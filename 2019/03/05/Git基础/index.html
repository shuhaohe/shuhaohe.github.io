第三节 Git的基本原理

举个例子，你新建了一个文件夹LearnGit，然后在这里执行了git init。

在LearnGit文件夹内的文件，可能处于以下几种状态：

  • untracked，未标记，意思是没有把这个文件添加到版本管理当中，虽然它恰巧也在LearnGit这个文件夹下。
  • tracked，已标记。就跟上面相反啦，你能够用git管理的所有文件都必须是tracked的。如何将一个文件添加到接受git管理的版本仓库当中?用`git add xxx

对于tracked文件，它有三种不同的状态，或者说不同的阶段。分别是：

  • 处于working space当中，意思是正在被编辑的文件，但还没提交（不太严谨）的文件都处于这个状态。
  • 处于staging 缓存区当中。通过git add xxx除了可以将一个untracked变成tracked，还可以将文件从working space加入到staging。只有staging的文件能够最终提交给版本仓库。
  • repository 仓库。git commit 就是将staging里面的文件提交到这里。
